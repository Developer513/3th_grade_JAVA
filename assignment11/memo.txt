네트워크

TCP/IP
Transmission Control Protocol 
두 시스템 간에 신뢰성 있는 데이터의 전송을 관장하는 통신규약 

TCP 에서 동작하는 응용프로그램 사례 
e-mail, FTP, 웹(HTTP) 등 
SMTP

IP 인터넷 프로토콜
패킷교환 네트워크에서 송신 호스트와 수신 호스트가 데이터를 주고 받는 것을 관장하는 프로토콜 
TCP 보다 하위 레벨 프로토콜

IP 주소 
네트워크 상에서 유일하게 식별 될 수 있는 컴퓨터 주소 
숫자로 구성된 주소 
192.156.11.15
각 자리는 8비트  총 32비트 주소 
숫자로 된 주소는 기억하기 어려우므로 www.google.com 과 같은  문자열로 구성된  도메인으로 사용
(DNS Domain Name Service) 문자주소를 숫자IP로 바꿈
현재는 32비트 주소 고갈로 128비트 IP 사용이 증가하는 추세이다. 

포트 
통신하는 프ㅗ그램 간에 가상의 연결단 포트 생성 
IP 주소는 네트워크 상의 컴퓨터 또는 시스템을 식별하는 주소 
모든 응용로그램은 하나 이상의 포트 생성가능
포트를 이용하여 상대방의 응용프로그램과 데이터 교환

잘 알려진 포트 
시스템이 사용하는 포트번호 (시스템에서 사용하는 포트번호)
0 ~ 1023
80 HTTP
21 FTP
23 텔넷
잘 알려진 포트번호는 충돌가능성이 있기 때문에 사용하지 않는것이 좋다. 

소켓 프로그래밍
소켓? TCP/IP 네트워크(인터넷)를 이용하여 쉽게 통신프로그램을 작성하도록 지원하는 기반기술
소켓 
두 응용프로그램 간의 양방향 통신 링크의 한쪽 끝단 
소켓끼리 데이터를 주고받음
소켓은 특정 IP포트 번호와 결합 
자바로 소켓 통신할 수 있는 라이브러리 존재
소켓종류 : 서버소켓, 클라이언트 소켓 


소켓을 이용한 서버/클라이언트 채팅 예제
서버와 클라이언트가 1:1로 채팅
클라이언트와 서버가 서로 한번씩 번갈아 가면서 문자열 전송 
문자열 끝에"\n 을 덧붙여 보내고 라인 단위로 수신
클라이언트가 bye 를 보내면 프로그램 종료
